## 1. 产品概述

### 1.1 产品定位

PengIP Desktop 是一款**三层订阅型 AI 助手桌面应用**，专为医美行业从业者设计。

**核心定位：不是卖软件，而是卖"工具 + 打法 + 内容"的完整解决方案**

**三层订阅模型：**

```
┌─────────────────────────────────────┐
│  内容订阅（表层）                    │
│  - 爆款模板、成功案例、营销话术      │
│  - 每周更新                         │
└─────────────────────────────────────┘
           ↓ 基于
┌─────────────────────────────────────┐
│  打法订阅（中层）                    │
│  - 方法论、策略、流程                │
│  - 算法规则、趋势分析                │
│  - 每月更新                         │
└─────────────────────────────────────┘
           ↓ 需要
┌─────────────────────────────────────┐
│  工具订阅（底层）                    │
│  - 专用工具、AI 能力                │
│  - 配合打法的技术实现                │
│  - 持续优化                         │
└─────────────────────────────────────┘
```

**提供：**
- AI 对话式交互
- **工具层**：多种专业工具（图片生成、视频生成、数据分析等）
- **打法层**：持续更新的方法论和策略（发布时间、互动技巧、算法规则）
- **内容层**：持续更新的爆款模板、成功案例、营销话术
- 本地快速执行
- 最小化服务器依赖

### 1.2 产品愿景

成为医美行业从业者的**必备新媒体营销助手**，通过订阅获得完整的解决方案：

**工具层（底层）：**
- 小红书图文生成器
- 抖音视频剪辑器
- 数据分析看板
- 私域运营工具

**打法层（中层）：**
- 小红书算法规则和应对策略
- 抖音流量池机制和投放技巧
- 私域转化流程和话术设计
- 跨平台内容分发策略

**内容层（表层）：**
- 最新的爆款模板
- 成功案例的复制方案
- 验证过的营销话术
- 行业趋势分析

**让用户获得的不只是工具，而是"工具 + 打法 + 内容"的完整体系。**

### 1.3 核心特点

- **内容订阅**: 持续更新爆款模板、话术、案例（核心价值）
- **热更新**: 无需重装，自动获得最新"打法"
- **本地优先**: 大部分计算在本地执行，快速响应
- **AI 驱动**: 通过对话完成任务，无需记忆复杂操作
- **开箱即用**: 所有依赖预装，安装后立即可用
- **可扩展**: 基于 Skill 系统，可无限扩展新功能
- **隐私安全**: 数据不离开本地，保护用户隐私

**医美从业者**

- 用户: 医美咨询师
- 需求: 制作小红书/抖音内容

- 用户: 医美机构运营

## 5. 功能需求

### 5.1 核心功能

#### 5.1.1 AI 对话

**功能描述**
- 用户通过自然语言与 AI 对话
- AI 理解用户意图，调用相应的 Skill
- 支持多轮对话，收集必要参数

**交互流程**
```
用户: 帮我生成一个双眼皮术后效果图
AI: 好的，请上传术前照片
用户: [上传照片]
AI: 请问是术后第几天的效果？
用户: 第 7 天
AI: [调用 HealVision Skill] 正在生成...
AI: 完成！这是效果图 [显示图片]
```

**技术要求**
- 调用 Claude 4.6 API
- 支持工具调用（Tool Use）
- 本地缓存对话历史
- 支持文件上传（图片、PDF 等）

#### 5.1.2 Skill 系统

**Skill 定义**
- 每个 Skill 是一个独立的功能模块
- 包含：名称、描述、输入参数、执行逻辑
- 可以是 Python 脚本、Node.js 模块、或 API 调用

**已规划的 Skills**

| Skill 名称 | 功能描述 | 输入 | 输出 | 积分消耗 |
|-----------|---------|------|------|---------|
| HealVision | 整形术后效果图生成 | 术前照片、手术类型、天数 | 效果图 | 10 |
| MotionX | AI 视频生成 | 提示词、参考图 | 视频 | 50 |
| PreVSim | AI 图文生成 | 提示词 | 图片+文字 | 5 |
| PDF to PPT | PDF 转 PPT | PDF 文件、风格 | PPT 文件 | 20 |
| Image Enhance | 图片增强 | 图片 | 增强后的图片 | 5 |
| Video Edit | 视频剪辑 | 视频、剪辑指令 | 剪辑后的视频 | 30 |

**Skill 扩展**
- 开发者可以添加新的 Skill
- 只需编写 Python 脚本 + 注册到系统
- 无需修改客户端代码

  - 发布建议（基于打法）
  - 互动脚本（基于内容库）
  - 数据追踪（基于工具）

**订阅价值：**

用户订阅后获得的不是单独的工具、打法或内容，而是：
1. **工具**：配合打法的专用工具
2. **打法**：基于算法规则的方法论
3. **内容**：验证过的模板和案例

**三者缺一不可，形成完整的解决方案。**

**更新频率：**
- **工具层**：持续优化（每月）
- **打法层**：算法变化时立即更新（每月）
- **内容层**：每周更新（新模板、新案例）

**竞争壁垒：**
- 工具可以复制
- 但"工具 + 打法 + 内容"的完整体系难以复制
- 需要持续的行业研究和内容创作

### 6.3 Skill 开发规范

**Skill 脚本模板**
```python
# scripts/example_skill.py
import sys
import json

def execute(params):
    """
    执行 Skill 的主函数

    Args:
        params: dict, 输入参数

    Returns:
        dict, 执行结果
    """
    # 1. 参数验证
    required_params = ['param1', 'param2']
    for param in required_params:
        if param not in params:
            raise ValueError(f"Missing required parameter: {param}")

    # 2. 执行逻辑
    result = do_something(params)

    # 3. 返回结果
    return {
        "success": True,
        "data": result,
        "message": "执行成功"
    }

if __name__ == "__main__":
    # 从命令行参数读取输入
    params = json.loads(sys.argv[1])

    try:
        result = execute(params)
        print(json.dumps(result))
    except Exception as e:
        print(json.dumps({
            "success": False,
            "error": str(e)
        }))
        sys.exit(1)
```

### 11.1 核心商业模式：三层订阅

**不是卖软件，而是卖"工具 + 打法 + 内容"的完整解决方案**

**用户付费获得：**

- 方法论和策略
- 算法规则和应对方案
- 流量池机制和优化技巧
- 例如：发布时间策略、互动技巧、标签选择方法

**第三层：内容订阅（表层）**
- 爆款模板
- 成功案例
- 营销话术
- 趋势分析
- 例如：3月最新爆款模板、验证过的标题库、成功案例拆解

**三层协同，形成完整的解决方案。**

**类比：**
- Netflix：订阅获得影视内容
- Spotify：订阅获得音乐内容
- **PengIP：订阅获得"工具 + 打法 + 内容"的完整体系**

## 12. 风险和挑战

### 12.1 技术风险

**风险 1: Python 环境打包**
- 风险: 跨平台兼容性问题
- 缓解: 提前测试，使用成熟的打包工具

**风险 2: 客户端体积**
- 风险: 1GB 体积可能影响下载
- 缓解: 提供增量更新，优化打包

**风险 3: AI API 稳定性**
- 风险: Claude API 可能不稳定
- 缓解: 实现重试机制，提供降级方案

### 12.2 商业风险

**风险 1: 内容创作压力**
- 风险: 需要持续创作高质量内容
- 缓解: 建立内容创作团队，建立内容库

**风险 2: 用户期望管理**
- 风险: 用户期望每周都有"爆款"
- 缓解: 明确说明是"方法论"，不是"保证爆款"

**风险 3: 竞争**
- 风险: 市场上可能出现类似产品
- 缓解: 持续的内容创作是护城河，快速迭代

**风险 4: 用户留存**
- 风险: 用户可能觉得"学会了就不需要了"
- 缓解: 强调"持续更新"的价值，算法和趋势在变化
